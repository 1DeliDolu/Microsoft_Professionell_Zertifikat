
### 1. Question 1

What is the primary purpose of a JOIN clause in SQL?

* [ ] To calculate summary statistics across columns
* [ ] To sort rows within a single table alphabetically
* [ ] To delete rows from multiple tables based on a common value
* [X] To combine rows from two or more tables based on a related column

That’s correct! The primary purpose of a JOIN clause in SQL is to combine rows from multiple tables based on a common column between them.

---

### 2. Question 2

Which type of JOIN returns all rows from the left table and the matching rows from the right table, with NULL values for non-matching rows?

* [ ] RIGHT JOIN
* [ ] INNER JOIN
* [ ] FULL OUTER JOIN
* [X] LEFT JOIN

That’s correct! A LEFT JOIN returns all rows from the left table and matches rows from the right table, filling NULL values for non-matches.

### 3. Question 3

How does a primary key relate to using JOIN clauses in SQL?

* [ ] It combines tables into a single dataset
* [ ] It sorts data in ascending order
* [ ] It restricts data to show only distinct rows
* [X] It acts as a unique identifier, allowing tables to be linked accurately

That’s correct! Primary keys uniquely identify rows, enabling accurate connections between tables in a JOIN.
